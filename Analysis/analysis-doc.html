1Introduction

Pong Tracker provides Users with a centralized website for recording and viewing statistics of Beer Pong games. These games are played by themselves and other Pong Tracker users. Details of each game are entered into the website as users are playing in real time. This analysis document details how the Pong Tracker product will work. It provides models on long-term information, user interfaces, and the associations between them.

2Purpose and Uses

The purpose of Pong Tracker is to provide a service to Users that is simple, intuitive and attractive, so that Users can gain from competition, without the complexity of managing the system themselves. Pong Tracker provides:

  • Users
    • Ability to create and score Games
    • Ability to track personal statistics
    • Ability to view other Users' statistics
    • Ability to view a leaderboard
  • Guests
    • Ability to participate in a Game
    • Ability to view a leaderboard
  • Administrators
    • Ability to manage Institutions
    • Ability to ban and unban Users

3Object Classification

3.1Entity Objects

Game A Game being recorded using Pong Tracker. A Game has two Teams.
Cup A Cup used in a Game on a Team. Each team has six Cups.
Team A Team participating in a Game. A Team has one to two Users, and zero to one Guests.
LifeStats The cumulative statistics collected and stored for a User or Guest over time.
Event An individual statistic collected and stored for a User or Guest.
Institution The Institution a User is enrolled in.
Administrator The actor named Administrator who has special permissions within Pong Tracker.
User The actor named User.
Guest The actor named Guest.

3.2Boundary Objects

Registration View Where a Guest registers to become a User.
Create Game View Where a User creates a new Game for simulation.
Score Game View Where a User scores a Game.
End Game View Where a User selects the winning Team, and under what circumstances the win was obtained.
Confirm Game View Where a User approves or denies the validity of the Game.
Rules View Where a User can read the rules of Pong Tracker.
LeaderBoard View Where a User can see the top ranked Users.
Statistics View Where a User sees their own or other Users' statistics.
Search User View Where a User can search for other Users.
User Login View Where a User logs into Pong Tracker.
User Logout View Where a User logs out of Pong Tracker.
Delete Account View Where a User may delete their account from Pong Tracker.
Limited LeaderBoard View Shows the top ten Users on Pong Tracker.
Administrator Login View Where an Administrator logs into Pong Tracker.
Administrator Logout View Where an Administrator logs out of Pong Tracker.
Delete Institution View Where an Administrator deletes an Institution.
Add Institution View Where an Administrator adds an Institution.
Ban User View Where an Administrator bans a User.
Unban User View Where an Administrator removes a ban from a User.

3.3Control Objects

End Game Controller Coordinates the actions involved with ending a Game.
LeaderBoard Controller Retrieves User data to display on LeaderBoard View.
Statistics Controller Coordinates retrieving and recording Stats about a User.
Search User Controller Retrieves User data on other Users. Used for looking at other User's statistics.
User Login Controller Retrieves User data to verify identity.
User Logout Controller Terminates User session on Pong Tracker.
Delete Account Controller Coordinates deleting a User.
Create Game Controller Coordinates creating a Game.
Score Game Controller Coordinates updating a Game.
Confirm Game Controller Coordinates confirming validity of a Game.
Register Controller Coordinates adding a User.
Administrator Login Controller Retrieves Administrator data to verify identity.
Administrator Logout Controller Terminates Administrator session on Pong Tracker.
Delete Institution Controller Coordinates the actions involved when an Administrator deletes an Institution.
Add Institution Controller Coordinates the actions involved when an Administrator adds an Institution.
Ban User Controller Coordinates the actions involved when an Administrator bans a User.
Unban User Controller Coordinates the actions involved when an Administrator removes a ban from a User.
