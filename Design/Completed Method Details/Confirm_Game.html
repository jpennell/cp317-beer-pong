Class Name ConfirmGameController
Method Name confirmOrDenyGame
Method State Public
Return Type None
Input Arguments int gameID, boolean isUserConfirming
Output Arguments none
Error Messages If no Game with the gameID exists:
***Game does not exist.***
If the Game has already been confirmed or denied:
***Game has already been confirmed/denied.***
If the Game has not actually ended yet:
***Game has not ended***
External Accesses Game, Event, EventType
External Changes If successful, the Game's isConfirmedOrDenied flag is set to True.
Other Methods Invoked obtainGame, isConfirmedOrDenied, isEnded, confirmGame, setConfirmedOrDenied
Description Using the gameID, obtains the Game with the obtainGame Utility function.
A valid Game is one that exists, hasn't already been confirmed/denied and has ended

If the Game is valid and the User is confirming the Game, confirmGame is called to update the players' LifeStats and Ranking values.
If a valid Game is being denied, no changes are made to anyone's LifeStats but the Game is still kept on record.
Successfully confirming/denying a Game results in its isConfirmedOrDenied flag being set to True.
