Input Arguments String username, String password
Output Arguments none
Error Messages If the entered password is incorrect:
***Incorrect password.***
External Accesses User
External Changes May remove the User's entry from the User table.
Other Methods Invoked authenticate, logout, setBanned
Description Obtains the User's username from their browser and input password from the HTML form then
checks to see if the User has correctly entered their password by calling Django's authenticate function.

If so, the User is logged out, their account is deactivated (not actually deleted,) and they are redirected to the
Pong Tracker homepage. Otherwise, they are brought back to the account deletion page with the error message.
Note: The User's account is not actually deleted so that any references made to it (i.e. past games) can still function.

Class Name CreateGameController
Method Name createGame
Method State Public
Return Type Game
Input Arguments String player1, String player2, String player3, String player4
Output Arguments none
Error Messages If the same username was entered multiple times:
***All 4 players must be distinct.***
If one of the usernames corresponds to an account that does not exist:
***Not all 4 players exist***
If one of the usernames corresponds to an account that has been banned:
***One of the 4 players is banned and thus cannot participate***
External Accesses Game, Team, User
External Changes If successful, new entries are added to the Game and Team tables
Other Methods Invoked arrayHasDuplicates, obtainUser, initializeNewGame
Description Obtains the 4 input player usernames from the HTML form.
They are placed in an array which is checked for duplicates by the Utility function arrayHasDuplicates.
An attempt to obtain the User objects that correspond to those usernames by calling the obtainUser Utility function
is then made.
If all 4 input usernames correspond to distinct valid User objects, the helper method initializeNewGame
is called and a new Game is created.

Upon success, the User is brought to the new Game's score game page.
If an error occurred, they are brought back to the create game page with the error message.

Class Name CreateGameController
Method Name initializeNewGame
Method State Private
Return Type Game
Input Arguments User user1, User user2, User user3, User user4
Output Arguments none
Error Messages none
External Accesses Game, Team
External Changes new entries are added to the Game and Team tables
Other Methods Invoked none
Description Creates two Teams with user1 & user2 on one Team, and user3 & user4 on the other.
These two Teams are used to create a new Game which is then returned.
