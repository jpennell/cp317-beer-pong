1Introduction

1.1Document Overview

This is the Design document for Pong Tracker, an online website which allows users to track their Pong playing skills and compare them to others. This is part of the project documentation for CP317 during the Spring 2013 class.

The first part is an overview of the governing guidelines for this project. The second part is about the logical design involving the architecture of Pong Tracker and how various elements interact. The contents of each package is made explicit as is the database which the project will use. The third part covers the technical design being the methods which are to be created. The fourth part will cover Django, the predominent tool which will be used in implementation. The fifth part will act as an introduction to some front end prototyping concluding with the final parts being references and SQA Version History.

1.2Target Audience

This document is for the technical team as it contains detailed information regarding the implementation procedures.

1.3Guidelines

1.3.1Technical Guidelines

Pong Tracker is hosted on a Sun Microsystem Linux Server with a MySQL database. The Linux server must be able to render HTML 5, CSS 3, Python 2.7, and JavaScript. The server is also required to integrate with the Django web framework and Github. Django and python were selected as the development team has python experience. Django is a strong web framework with many tools built in such as sessions, and database APIs. The only concern is the implemention is using python 2.7 when the current version is 3.3.2. 2.7 has been selected because Django support for 3.3.2 is currently only an experimental version.

Coding Guidelines and Conventions:

  • Must meet HTML 5 and CSS 3
  • Must be coded in python 2.7 excluding JavaScript
  • Must be object-oriented
  • Variables must be named using Camel Case

Testing:

  • All functions are unit tested.
  • Acceptance testing by the client is necessary.

1.3.2Ethical & Legal Guidelines

  • The application will make use of libraries created by third-party developers; however, to respect intellectual property rights, the application will be developed using libraries under open source licenses.
  • All coding must be accompanied by comments.

The utilities package includes helper functions that are not specific to a particular package (User, Game, Statistics) in our application but are accessed by some or all of them.
