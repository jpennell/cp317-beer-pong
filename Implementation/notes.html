Administrator Setup and Notes

Set Up:

  1. In the settings file you will need to set the following fields to meet any changes.
    • ADMINS: The admins for the website
    • DATABASE: The database to use
    • ALLOWED_HOSTS: The domains that can run the code
    • EMAIL_HOST
    • EMAIL_HOST_USER
    • EMAIL_HOST_PASSWORD
    • EMAIL_PORT
    • EMAIL_USE_TLS
    • SITE_URL: This is required when part of the url gets truncated, a "/" may suffice depending on server configuration
    • MEDIA_ROOT: The location to upload profile photos
    • STATIC_URL: Where the static files are
    • STATICFILES_DIRS: names of the directories of static files
  2. Administration will also need to run ./manage.py syncdb and create a superuser
  3. Please create the the corresponding rank rows for the super user
  4. Load the event types ./manage.py loaddata fixtures/eventtypes.json
  5. Load the universities you want ./manage.py loaddata fixtures/institutions.json
  6. Run the RankView
  7. If you have a database and want to keep the data use ./manage.py dumpdata --indent=4 app.model > fixtures/filename.json
  8. If you want to update your code on an apache server run: touch wsgi.py
  9. There are also some instances of the email in the code please feel free to change.

Notes:

  1. If there an issue arises about Institution and admin, check the User admin file.
  2. Testing files no longer work as functions have been moved, they are kept for reference.
