    Product Functions

    1. A user friendly, web-based application (computer and mobile) in which users can create and score beer pong games while they are being played
    2. Must have the ability to show personal statistics regarding wins, losses, cups sunk, and preferred partner
    3. Must have the ability to show which end user is the best beer pong player using ranking system defined in section (insert section reference here)

    Actors and Use Cases

    1. Administrator - is an individual who has special privileges within the website. An administrator can login, logout, add an institution, delete an institution, ban a user, and unban a user. Administrators and end user accounts are completely separate.
    2. End User - is an individual who uses the website for active game play. In terms of game play, the end user may create, score or confirm a game. End users can view the entire leaderboard, login, logout, inactivate their account, view their personal statistics, and view other players’ statistics.
    3. Guest - is an individual who is not using the site for active game play. A guest may view a limited version of the leaderboard and may register. Once a guest has registered, they become an end user.
    1. Admin Login - an administrator must provide their username and password as discussed in section (reference to section here) to authenticate their identity
    2. Admin Logout - an administrator need only select the “Logout” option to have access to their account session terminated
    3. Add Institution - an administrator has the ability to add institutions that guests select when registering to become a user.
    4. Delete Institution - an administrator has the ability to delete institutions that guests select when registering to become a user.
    5. Ban User - an administrator can ban a user who has not conformed to the site’s rules or policies
    6. Unban User - an administrator can lift a ban on a user who has not confirmed to the site’s rules or policies
    7. User Login - an end user must provide their username and password as discussed in section (reference to section here) to authenticate their identity
    8. User Logout - an end user need only select the “Logout” option to have access to their account session terminated
    9. Create Game - an end user can create a beer pong game
    10. Score Game - once a beer pong game has been created, an end user uses the interface to record the results of the game as it is being played
    11. Confirm Game Results - once a game has been completed, an end user must confirm the results of the game in order for the game data in question to be used and considered in the statistics and leaderboard areas
    12. View Personal Statistics - an end user may view their personal statistics at any time given they are logged in. The statistics shown here include only games that have been confirmed by players involved in a particular game.
    13. View Other Users’ Statistics - an end user may view other players’ personal statistics at any time given they are logged in. The statistics shown here include only games that have been confirmed by players involved in a particular game.
    14. View Full Leaderboard - an end user may view the entire leaderboard at any time given they are logged in. The statistics shown here include only games that have been confirmed by players involved in a particular game.
    15. Register - a guest must provide information as stated in section (reference to section here) in order to facilitate the login functionality
    16. View Limited Leaderboard - a guest can see the top ten players on the website. For full access to the leaderboard, a guest must become a user

    User interface

    • The user interface for the beer pong network is a dynamic website.
    • Users log in, and the content changes according to their account level.
    • Users will see a front page with their current statistics. They have the ability to create a new game.
    • The game interface will be entirely visual, to simulate the experience of a Pong game.
    • The function of the interface is to input data about the game into the database as easily and intuitively as possible.
    • Mobile Component
      • In order to make data entry more accessible, we hope to implement a mobile-ready version of the web user interface
      • The mobile website will be a simplified version of the main website — but it must be able to fully support game data entry

    Software interface

    1. The system operates on standard internet browsers (Internet Explorer 6/7/8/9/10, Firefox, Google Chrome, Safari).
    2. The system uses a database.
    3. Analytic Algorithm
      • The system implements a library which contains an analytic algorithm
      • The library allows us to specify what inputs we give the algorithm which then calculates the skill level in comparision to other users.
      • Some documentation is available here GitHub TrueSkill Readme, TrueSkill Site and here Elo Rating System Documentation
      • We are permitted to use the algorithm
    4. OpenID
      • The system allows a user to login with an OpenID account
      • Supported providers are Google and FaceBook.
      • The library allows us to grab a user's profile information from either an active session or by redirecting them to their providers login page
      • Some documentation is available here GitHub Ruby-OpenID and here The OpenID Site
