Pong Tracker CP317 2013

The Tasks

For the project, the class must produce the following:

  • A requirements document listing the product requirements (The course instructor is the project client): Requirements Document
  • A rapid prototype of the product GUI
  • An informal specification document: Contained in the Requirements Document
  • An object-oriented analysis: Analysis Document
  • An object-oriented design: Design Document
  • An UML diagnram: UML report
  • The implementation, including:
    • Coding Standards
    • actual coding: in the pongtracker folder
  • A project timeline- Henry
  • User documentation - Carlie
  • Documentation standards: PEP-8 standards
  • Student particpation records:
    • Wiggio Calendar
    • Trello

All diagrams and models must be produced using UML (Unified Modeling Language). Tools for all aspects of the product development will be provided by the course instructor.

Each student is expected to contribute to a majority of these items, either by writing or reviewing individual elements. A student's project mark is determined by the quality of their contribution. The project is worth 45% of each student's final grade.

The project and its progress will be discussed in each lecture and used an example in the class discussion of the various elements of software engineering.
